# Support

## Best Path By Topic

- Usage questions, operational questions, and maintainer contact: `minecraft-ping@omkhar.net`
- Bugs and regressions: open a GitHub issue with the bug report template
- Feature requests: open a GitHub issue with the feature request template
- Security vulnerabilities: follow [SECURITY.md](SECURITY.md) and report privately instead of using public issues

## What To Include

When asking for help by email, include:

- the `minecraft-ping -V` output or the exact commit
- the command you ran
- the target edition and address family when relevant
- the observed output or error
- the operating system and architecture

## Expectations

Support is best-effort.
Public issues and pull requests are preferred for bugs and feature work so the repository keeps an auditable history.
Security reports should stay private until a fix and disclosure plan are ready.
